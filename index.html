핵심 개념 정리

  • AI Agent — 주어진 목표를 달성하기 위해 스스로 판단하고 행동하는 AI 시스템
  • 멀티테넌트 — 여러 사람이 같은 시스템을 공유하는 구조
  • 감사 로그 — AI가 사용자 대신 행동한 기록
  • Zero Data Retention (ZDR) — 데이터를 보관하지 않음
    • 학습 ≠ 데이터 보관 → 각각 다른 리스크 존재
    • 학습: 데이터 유출 가능성 / 보관: 해킹·내부 직원 리스크

Part 1 — 3

OpenClaw, 왜 문제인가?

네이버·카카오·당근이 OpenClaw를 금지한 이유
  • 오픈소스, 비공식 제품
  • AI API의 판단을 그대로 실행하는 "집행관" 역할
  • 안전장치 없음 → 위험한 명령도 그대로 실행

상용 프로그램과의 차이

  • 위험 명령 차단
  • 사용자 승인 필요
  • 실행 범위·횟수 제한
Part 1 — 4

Claude Code 취약점, 정말 위험한가?

  • 발견된 취약점 3개: 오픈소스 사용 또는 내부 직원 악의적 공격 시나리오
  • 실제 피해 사례 없음
  • Check Point Research와 협력 → 패치 완료
"AI 도구가 특별히 위험한 게 아니라,
자동화 범위가 넓어진 만큼 설정 파일 하나의 영향력이 커졌다"
  • VS Code 플러그인, npm 악성 패키지와 동일한 종류의 위협
Part 1 — 5

안전하게 쓰는 방법

  1. 오픈소스 사용 제한 — 신뢰도 높은 오픈소스만, 사전 검증 후 사용
  2. MCP 사용 제한 — 오픈소스와 동일 기준 적용
  3. 공식 AI 서비스 선택 — ChatGPT, Claude, Gemini 등
  4. 피해 범위 격리 — 홈 디렉토리 작업 ✕, 최소 권한 IAM 사용
  5. 신규 기능 사전 검토 — 업데이트 내용 파악 후 사용
  6. AI 제안 명령어 직접 검증 — 명령과 옵션 이해 후 실행
Part 1 — 6

결론

  • Claude Code ≈ Cursor (OpenClaw와는 다른 선상)
  • "특정 서비스 금지"보다 "신규 AI 서비스 사용 전 검토" 규칙 수립이 효과적

권장 방향

  • AI 에이전트 사용 허용
  • AWS IAM 등 계정 권한 제한
  • 오픈소스 사용 전 검토 프로세스
  • 비즈니스 플랜으로 관리자 모니터링 환경 구성
Part 1 — 7

AI 대상 공격 유형

챗봇 서비스(H.AI 등) 대상

  • 프롬프트 인젝션
    악의적 지시를 주입해 의도치 않은 동작 유발
  • AI 메모리 포이즈닝
    장기 메모리에 허위 정보 주입 → 이후 응답에 지속 영향

→ 악의적 요청 검증 필요, 민감 데이터 연결 ✕

개발자 대상

  • 프롬프트 인젝션
    외부 콘텐츠를 AI 소스로 줄 때 공격 가능
  • 할루시네이션 스쿼팅
    AI가 존재하지 않는 패키지명 추천 → 악성 패키지 설치 유도

→ 외부 콘텐츠 AI 입력 주의, 오픈소스 사용 전 검토
